# Canvas 交互与 Agent 设计规约

## 1. 这次重捋要解决什么

当前问题不只是“文案不自然”，而是三层心智模型没有对齐：

- 用户以为自己在和一个创作搭档共创，但系统经常突然切成流程机。
- 对话以为自己是主轴，画布却还在承担候选方案和阶段跳转，导致决策位置摇摆。
- Agent 表面上有多个角色头像，实际上只是按阶段换皮，没有真实分工，所以像“演 agent”。

这份设计把基线改成一句话：

**用户始终只在和一个“主创搭档”对话；系统内部可以有分工，但对外只暴露一个连续、稳定、自然的协作体验。**

## 2. 产品定位

Canvas 不是表单，不是 BPM 流程图，也不是让用户手搓节点的工作台。

Canvas 的正确定位是：

- **对话是主线**：所有关键决策都在聊天里发生。
- **画布是镜子**：只负责把已经形成的内容关系可视化。
- **节点优化是局部编辑**：用户点节点时，是在当前作品里做局部改写，不是跳到另一套流程。

一句更直白的话：**先聊清楚，再生成；生成完再上画布；画布负责看见，不负责拍板。**

## 3. 新的对话创作主流程

整条链路不再按“问字段 -> 选项 -> 下一步”推进，而是按一个反复循环的创作节奏推进：

`理解 -> 收敛 -> 提案 -> 确认 -> 执行 -> 回看`

每个阶段都遵守这个节奏，只是聊的对象不同。

### 3.1 阶段一：立题与创意收敛（idea）

目标不是收集完所有字段，而是先把“这到底是个什么片子”说清楚。

这一阶段优先锁定 4 件事：

- 题材和核心冲突
- 风格和情绪基调
- 预估时长与节奏密度
- 面向谁看、希望观众带走什么感受

对话原则：

- 不逐项盘问。
- 一轮只推进一个关键不确定点。
- 当信息已经够用时，直接提案，不再继续问。

合格的体验应该像：

- 用户给一个模糊想法。
- Agent 先复述理解，再给 2-3 个方向。
- 用户挑一个，或在其上微调。

而不是：

- “请问类型是什么？”
- “请问风格是什么？”
- “请问时长是什么？”

### 3.2 阶段二：剧本方向提案与定稿（script）

这里不应该让用户从零写剧本，而应该先给“方向性方案”。

推荐节奏：

1. Agent 基于已收敛的创意，给出 2-3 个剧本方向。
2. 每个方向都带一句“为什么这样走”。
3. 用户选择一个方向，或说“拿 A 的冲突、B 的结尾”。
4. Agent 整合后形成当前定稿剧本。

这一阶段的核心不是“生成一段长文本”，而是**帮助用户做方向决策**。

所以候选方案必须：

- 在对话里展示，不丢给画布。
- 有差异，不是同义改写。
- 有取舍说明，不只是标题党。

### 3.3 阶段三：分镜拆解（storyboard）

剧本确定后，不要立刻进入“开始分镜吧”的命令式状态。

更自然的节奏是：

1. Agent 先说清这版剧本适合怎样的镜头组织。
2. 必要时先确认整体画面方向，而不是上来逐镜生成。
3. 生成完成后，引导用户只修改“有问题的镜”，而不是重新走全流程。

这一阶段的主问题是：

- 画面节奏顺不顺
- 哪些镜头需要强调
- 哪些对白需要留白或改成画面表达

换句话说，storyboard 阶段的对话重心应该从“收集信息”变成“导演视角的调整建议”。

### 3.4 阶段四：配音匹配（voice）

配音不是单纯的一键后处理，而是一次“表演层”的确认。

这一阶段先聊的是：

- 主角色应该更克制还是更有戏
- 旁白要不要明显存在感
- 整体语速和情绪起伏

只有这些判断已经明确，用户又表达“可以开始了”，才触发配音匹配。

### 3.5 阶段五：成片渲染与回看（video）

渲染前不该像提交表单一样“最后一步，点生成”。

更好的动作是：

1. Agent 先做一次简短回顾：当前片子是什么、节奏如何、已锁定哪些风格判断。
2. 告诉用户“如果现在出片，会是什么感觉”。
3. 用户确认后再触发渲染。

渲染后，Agent 的职责不是宣布结束，而是引导用户做最后一轮回看：

- 开头够不够抓人
- 中段会不会拖
- 结尾停顿够不够

## 4. 对话应该怎么变得不生硬

## 4.1 每一轮只做一种事

一条消息只能主要承担下面四类职责之一：

- `理解`：确认自己听懂了什么
- `提案`：给出方向或建议
- `确认`：让用户拍板
- `推进`：在条件满足时执行动作

不要一条消息里同时做三件事，比如：

- 既追问、又总结、又催下一步

这会让用户感觉自己在被流程推着走。

## 4.2 先回应，再提问

自然感的关键不在“少问”，而在“先接住用户”。

推荐结构：

1. 用一句话复述理解或判断
2. 再给建议 / 选项 / 下一问

例如：

- “这个方向的亮点是反差感很强，适合做短平快的开场。要不要我给你拆成三个更明确的剧情走向？”

而不是：

- “请选择剧情走向。”

## 4.3 不把阶段名暴露成流程命令

用户不应该频繁看到：

- “进入分镜阶段”
- “进入配音阶段”
- “下一步请进行渲染”

阶段存在于系统内部，用户看到的应该是创作语言：

- “这版剧情已经稳了，我来把它拆成画面。”
- “镜头节奏差不多了，接下来可以把声音气质定下来。”

## 4.4 动作前必须有“确认感”

任何昂贵或不可逆的动作前，都要满足两个条件：

- 用户明确表达推进意图
- Agent 先用一句话说明“现在为什么可以推进”

例如：

- “现在人物关系和节奏都已经定住了，我可以按这版去出分镜。要我开始吗？”

这样用户会觉得是在合作，而不是被系统抢方向盘。

## 5. Agent 设计重构

## 5.1 对外保留一个主搭档，但要看得见当前实际工作的专家

用户可感知的身份应该只有一个：

- **主创搭档**

它始终保持同一种语气、同一种协作关系，对作品全局负责。

但“内部谁在实际工作”不能完全隐身，否则用户会觉得 agent 分工是假的。

因此前台展示建议改成：

- 始终以 **主创搭档** 作为主身份说话
- 在消息头或身份徽章上明确标出 `当前协作: 编剧 / 分镜导演 / 配音导演 / 后期剪辑`
- 专家身份是“当前工作台”的可见标签，不是一个新的对话人格

不建议把“编剧 / 导演 / 配音导演 / 剪辑师”做成四个对外轮流说话的人格，否则用户会有两个感受：

- 角色是为了 UI 热闹，不是真分工
- 每到一个阶段就像换了一个人接手，连续感被打断

## 5.2 内部分工可以保留，但应该前台统一、专家可见

更合理的设计是“单前台，多后场”：

- **主创搭档（前台 Orchestrator）**：唯一对外说话者，负责理解用户意图、组织提案、推进节奏
- **剧情顾问（后台）**：负责冲突、人物弧光、结构建议
- **分镜导演（后台）**：负责镜头组织、景别、节奏与画面可拍性
- **表演与声音顾问（后台）**：负责音色、语速、情绪强弱
- **制片检查器（后台）**：负责阶段完成度判断，决定是否具备执行条件

前台统一说话，后台提供判断依据；同时前台要让用户知道“这轮主要是谁在协作”。
这样既能保留专业分工，也不会把用户扔进“多 agent 角色扮演”。

## 5.3 Agent 的职责不是“问完”，而是“推进决策”

每一阶段 Agent 的 KPI 不是采集字段完整率，而是：

- 是否把关键不确定性收敛掉了
- 是否给了用户足够清晰的选择
- 是否在合适时机推进执行

这意味着 Agent 不该执着于问满所有字段，而要学会：

- 哪些信息已经够用了
- 哪些地方该直接给方案
- 哪些地方必须停下来确认

## 6. 画布与对话的职责边界

### 6.1 对话负责决策，画布负责呈现

明确规则：

- 候选方案只出现在对话里
- 画布只显示已经提交或已生成的内容
- 节点点击只打开局部优化，不改变主线决策位置

### 6.2 节点优化是“局部工位”，不是第二条主线

用户点某个节点后，系统不该把全局对话切去另一个状态机。

更合理的模型是：

- 主线对话还在
- 节点面板负责局部编辑
- 编辑结果回写后，主创搭档可以在主线里用一句话承接变化

例如：

- “这场对白已经收紧了，现在节奏更利落。要不要顺手把后两镜也统一一下语气？”

## 7. 与当前实现的对应关系

这套设计不是推倒重来，而是给现有实现定更清晰的方向。

### 7.1 已经对的部分

- `chat()` 已经成为全流程统一入口
- `AgentStateMachine` 已退化为阶段追踪器
- 剧本候选已经回到对话内卡片，而不是画布候选节点
- 画布已经在朝“镜子”而不是“主操作台”收敛

### 7.2 仍需要继续调整的部分

- 后端 prompt 还需要更强调“先回应再推进”的节奏
- 前端 agent 头像不该暗示“多个轮流接管的人格”
- 触发动作前的确认文案还可以更像共创，不像按钮代理
- 节点优化后的主线承接还比较弱，局部编辑和全局叙事感仍偏分离

## 8. 可执行的落地原则

之后无论改 prompt、前端消息结构还是 agent 编排，都遵守下面 8 条：

1. 用户始终面对一个稳定的主创搭档。
2. 一轮对话只推进一个关键决策。
3. 先回应用户，再提问或提案。
4. 信息够用时优先给方案，不继续盘问。
5. 候选方案留在聊天里，画布只放已提交内容。
6. 触发生成前必须有“为什么现在可以开始”的一句确认。
7. 节点编辑是局部修订，不另开第二套主线流程。
8. 每个阶段结束时，Agent 要做简短回看，而不是只宣布完成。

## 9. 一句话版本

如果只记一条：

**把 Agent 从“流程推动器”改成“决策搭档”，把画布从“候选承载层”改成“结果镜子”，整条漫剧生成流程自然感就会回来。**
