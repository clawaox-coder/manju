# 页面交互规格

## 1. 目标

这份文档定义 Canvas 页的具体交互分工：

- 左侧导演协作台做什么
- 中央视觉工作面做什么
- 右侧对象检查台做什么
- 一轮交互在三者之间怎么流动

目标不是画高保真，而是让任何人都能回答：

**用户此刻在这页上，到底应该去哪儿看、去哪儿改、去哪儿拍板。**

## 2. 页面结构

建议固定为 `顶栏 + 三栏主区`。

## 2.1 顶栏：制作驾驶舱

显示全局状态，不承载局部操作。

应包含：

- 项目标题
- 当前阶段
- 当前 bottleneck
- 当前已锁定核心方向
- 当前高优先级待确认数
- 当前风险数
- 最近一次生成状态

顶栏的作用只有一个：

**让用户一进来就知道项目卡在哪。**

## 2.2 左栏：导演协作台

左栏是决策层，不是普通聊天框。

它负责：

- 接住用户自由表达
- 解释当前焦点对象
- 比较候选差异
- 做确认
- 汇报对象级任务进度

它不负责：

- 承载所有主操作
- 替代右侧对象控制
- 变成纯消息流水

## 2.3 中栏：作品主舞台

中栏是主舞台。

它负责：

- 承载创作对象
- 展示候选与主线
- 支持对象选择、框选、合并、锁定
- 展示对象状态和联动影响

它不是：

- 对话回显区
- 只读结果区
- 抽象流程图

## 2.4 右栏：对象检查台

右栏只服务当前焦点对象。

它负责：

- 展示对象详情
- 调整对象参数
- 触发对象级生成
- 查看上下游影响
- 进行局部编辑

它不负责：

- 全局项目决策
- 阶段级推进

## 3. 左栏交互规格

## 3.1 左栏必须永远回答四个问题

左栏每一刻都要让用户看见：

1. 现在谁在协作
2. 现在在解决什么问题
3. 现在正在讨论哪个对象
4. 我下一步可以做什么

建议固定结构：

- 当前协作工种
- 当前任务标题
- 焦点对象标签
- 解释区
- 动作区
- 输入区

## 3.2 解释区

解释区不是泛泛聊天，而是焦点相关判断。

它的文案类型只应有 5 种：

- 解释当前对象
- 比较多个对象
- 说明状态变化原因
- 说明为什么现在可以推进
- 做生成后的回看

不建议出现：

- 大段无对象引用的泛聊
- 纯阶段口号
- 重复追问已锁定信息

## 3.3 动作区

左栏动作区不放通用大按钮，只放当前决策相关动作。

建议动作类型：

- `锁定这个方向`
- `比较这两个方案`
- `按这个对象生成`
- `接受风险继续`
- `重做当前对象`

动作区永远要挂靠当前焦点对象或当前 decision gate。

## 3.4 输入区

输入区要支持 3 种输入方式：

- 纯文本表达
- 粘贴/拖拽参考
- 基于焦点对象的补充说明

输入提示文案必须跟着焦点对象变。

示例：

- 焦点是 `style_direction`
  - “补充你想要的视觉气质或参考图”

- 焦点是 `storyboard_card`
  - “描述这一镜你想改哪里”

- 焦点是 `voice_strategy`
  - “补充你想要的语速、音色或旁白感觉”

## 4. 中栏交互规格

## 4.1 画布默认视图

默认不应该是空白无限画布。

第一屏应优先出现：

- 当前主线对象
- 当前活跃候选
- 当前高风险对象
- 当前待确认对象

也就是说，画布一打开就要有“问题密度”。

## 4.2 对象展示规则

对象在画布上至少要让用户一眼看出：

- 这是哪类对象
- 它现在是什么状态
- 它是不是主线
- 它是否有风险

建议基础视觉编码：

- 类型：对象形态/头部标签
- 状态：边框/角标/底色
- 焦点：高亮
- 风险：警示标
- 锁定：锁定标

## 4.3 基础操作

中栏必须支持这几种基础操作：

- 单击对象：设为当前焦点
- 双击对象：进入右栏编辑态
- 框选多个对象：进入比较态
- 拖拽对象：重组视觉结构
- 连接对象：建立引用关系
- 合并候选：生成新对象

其中最关键的是：

**单击不等于编辑，单击首先是建立共同语境。**

也就是说，单击对象后：

- 左栏解释这个对象
- 右栏显示这个对象
- 对话记忆切到这个对象

## 4.4 多选比较态

当用户框选多个决策对象时，页面进入比较态。

页面行为建议：

- 左栏切换为比较说明
- 右栏展示对象并排摘要
- 中栏高亮这批对象及其差异

左栏动作区改成：

- `比较差异`
- `合并优点`
- `锁定其中一个`

## 4.5 状态变化的可视反馈

对象状态变化后，中栏必须立即反映。

重点场景：

- `locked`
  - 出现锁定标

- `generating`
  - 出现处理中状态

- `warning`
  - 出现黄色风险提示

- `stale`
  - 显示过期/需重确认状态

这一步非常关键，因为它是对话结果的空间证据。

## 5. 右栏交互规格

## 5.1 无焦点态

当没有选中对象时，右栏不应该空着。

应显示：

- “请选择一个对象查看详情”
- 或显示当前主线对象摘要

## 5.2 单对象焦点态

当选中一个对象时，右栏展示：

- 对象标题
- 当前状态
- 上游依赖
- 下游影响
- 参数面板
- 局部生成/重做动作
- 历史修改

## 5.3 多对象焦点态

当是多选焦点时，右栏应切换成比较摘要，而不是强行展示一个对象。

建议显示：

- 对象列表
- 差异点
- 冲突点
- 可执行动作

## 5.4 对象级生成区

所有生成动作都尽量落在右栏对象级控制区。

示例：

- `基于当前剧本版本生成分镜`
- `重做这一镜`
- `按当前风格刷新角色形象`
- `按当前声音策略重跑配音`

这样用户会知道：

- 生成是挂在对象上的
- 不是神秘全局动作

## 6. 一轮交互的标准流

## 6.1 从画布发起

典型流程：

1. 用户点击一个对象
2. 中栏把对象设为焦点
3. 左栏切换到“解释这个对象”
4. 右栏展示该对象详情
5. 用户在左栏补充一句话，或在右栏直接执行对象级动作
6. 系统更新对象状态
7. 左栏解释结果

这条流体现的是：

**画布发起，对话承接，右栏执行。**

## 6.2 从对话发起

典型流程：

1. 用户在左栏说“我想让开头更抓人”
2. 系统根据 focus / canvas summary 判断当前相关对象
3. 中栏高亮 `storyboard_card: shot-03`
4. 左栏开始围绕 `shot-03` 给出判断
5. 右栏展示可重做项

这条流体现的是：

**对话提出问题，画布给出对象，右栏承接操作。**

## 6.3 从状态变化发起

典型流程：

1. 用户修改 `style_direction`
2. 系统发现 4 个分镜受影响
3. 中栏把这 4 个分镜标成 `warning` 或 `stale`
4. 左栏解释“哪些对象受影响了”
5. 右栏出现 `是否重刷受影响对象` 的对象级控制

这条流体现的是：

**联动先发生，可视反馈先到位，对话再解释。**

## 7. 各阶段的页面主节奏

## 7.1 创意期

页面重点：

- 中栏显示多个方向卡
- 左栏负责比较和解释
- 右栏负责查看方向细节与锁定

## 7.2 剧本期

页面重点：

- 中栏显示多个剧本版本或结构块
- 左栏负责说明戏剧收益和代价
- 右栏负责局部重写与锁定

## 7.3 视觉期

页面重点：

- 中栏显示角色、场景、分镜板
- 左栏负责说明对象关联和风格判断
- 右栏负责局部生成与替换

## 7.4 声音期

页面重点：

- 中栏显示声音策略与角色绑定关系
- 左栏负责解释声音选择的情绪后果
- 右栏负责替换和重跑

## 7.5 出片期

页面重点：

- 中栏显示版本输出与风险对象
- 左栏负责回看诊断
- 右栏负责回滚、开分支、重出

## 8. 页面上最重要的 5 条禁忌

### 禁忌 1

左栏说的内容不指向任何对象。

### 禁忌 2

右栏动作是全局动作，不知道到底作用在哪。

### 禁忌 3

中栏对象状态变了，但左栏没有解释。

### 禁忌 4

用户选中了对象，但对话仍在泛谈全局。

### 禁忌 5

用户在左栏做了决策，中栏却没有留下任何证据。

## 9. 第一版最小可行页面

如果要控制实现量，第一版建议先保证这几件事：

1. 顶栏能显示当前 bottleneck
2. 左栏能显示当前协作工种、当前任务、焦点对象
3. 中栏支持单击焦点、对象状态显示、基础多选
4. 右栏支持单对象详情和对象级生成按钮
5. 对话结果能让中栏对象状态立即变化

只要这五件事成立，页面就已经比当前体验提升一个层级。

## 10. 结论

这页最重要的不是三栏本身，而是三栏之间的职责边界：

- 左栏负责判断
- 中栏负责承载对象与事实
- 右栏负责执行对象级动作

如果三者职责清楚，用户会感觉自己在一个创作系统里。

如果三者职责混乱，用户就会觉得自己在三套互相抢戏的界面里来回跳。
